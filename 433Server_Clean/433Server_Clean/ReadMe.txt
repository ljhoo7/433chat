최적의 방법은 다음과 같다.

이 방법은 stdafx.h를 사용함에 따라서 프리 컴파일 헤더의 이점을 최대한 누릴 수 있도록 해준다.

그리고 #ifndef A #define A ~ #endif를 덕지덕지 써서 대서 꼬이는 문제를 겪지 않도록 해준다.

1. VS에서 기본적으로 제공하는 WIN API 솔루션 생성 마법사를 이용한다.

2. 인클루드하고 싶은 헤더들은 모두 stdafx.h에 기입한다.

3. 1 파일 1 클래스 원칙을 지키고 VS의 클래스 생성 마법사를 이용한다. 그 마법사 이용시 클래스 명 앞에 C를 붙여준다.

4. 마법사로 생성된 클래스의 소스 파일에 #include "stdafx.h"외에 #include "클래스명.h"도 있는데 이를 지워준다.

5. stdafx.h의 모든 #include 전처리기들보다 위에 현재까지 만든 모든 클래스들을 선언해준다. 즉, 클래스 몸통 없이 'class 클래스명;' 이런 식으로 모든 클래스의 이름을 컴파일러에게 사전에 알려줄 수 있도록 한다.

6. 전역 구조체는 상기의 5와 같이 #include들 앞에 선언을 해주고 몸통은 #include들 뒤에 써준다.
